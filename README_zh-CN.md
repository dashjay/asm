# ASM —— 家庭资产管理

> 其他语言：[English](README.md)

一个简洁、离线优先的**家庭净值追踪器**。每个周期花五分钟更新一次账户余额，就能跨成员、跨账户、跨币种地看清家庭财富的变化。

技术栈：Flutter、Drift（SQLite）、Riverpod、go_router、fl_chart。

---

## 它是什么（以及不是什么）

ASM 是一个**净值快照追踪器**，而不是流水账本。

- 它记录的是「**某时刻每个账户的余额**」，而不是每一笔交易流水。
- 核心价值循环是：**定期更新余额 → 看到家庭净值与走势 → 对资产状况心里有数。**
- 它的护城河是「**低门槛、家庭维度、多币种**」，而不是「记得全、记得细」。

为了保持简洁，刻意**不做**：

- 每日流水 / 双分录记账
- 预算信封与账单管理
- 银行卡 / 信用卡自动归集
- 社交功能与理财产品导购

每一个功能取舍都遵循同一条原则：**守护「5 分钟看清家底」的体验。** 产品的成败只系于一件事——**用户会不会持续回来更新余额**——因此最该盯的指标不是 DAU，而是 **「按周期完成余额更新的依从率」**。

## 现有功能

- **家庭与账户** —— 支持多个家庭成员，每人多个账户。
- **账户类别** —— 活期、固定资产、投资理财、负债、债权。负债会自动从净值中扣减。
- **多币种** —— CNY / USD / SGD，每次快照独立记录汇率，可统一设置展示币种。
- **余额快照** —— 更新向导一次性录入汇率与各账户余额；仪表盘会把同一天的多次更新合并为一个走势点。
- **净值走势图** —— 家庭总净值随时间变化，并带线性预测；详情页支持**按成员与资产类别任意组合的多选筛选**，使资产在类别间转移时仍能正确呈现。
- **首页仪表盘** —— 当前净值、较上次的变动、距上次更新天数、以及到期待更新的账户。
- **更新提醒** —— 每个账户可设置提醒周期，配合本地通知。
- **变动原因** —— 为每次余额变动打标签（工资、投资收益、还贷、购置资产、转账、赠予、汇率波动、其他）。
- **数据备份** —— 本地导出/导入，以及兼容 S3 的备份。
- **本地化** —— 支持英文、简体中文与日文，可在「设置」中切换。新增一门语言的成本很低，见[新增语言](#新增语言)。

## 产品路线图

以下从产品视角排定优先级，全部围绕「保住核心循环（让用户持续更新余额）」与「打造差异化」展开。

### P0 —— 保住核心循环 & 建立金融信任

1. **应用锁（PIN + 生物识别）。** 财务类应用的桌面级要求；目前家庭全部资产一目了然却没有锁屏，是信任硬伤。
2. **汇率自动获取（保留手动覆盖）。** 每次更新都手动录 USD→CNY / USD→SGD 是更新仪式里最大的摩擦点。改为联网拉取、一键填入、仍可手改，离线时回落到手动/默认值。降低更新摩擦 = 直接拉高更新依从率。
3. **最小输入的更新流程。** 默认带出上次余额、只改动过的账户；提供「本次无变化」一键沿用；在更新流程内高亮「尚未更新的账户」。
4. **可靠的自动备份与放心恢复。** 定时备份、显示「上次备份时间」、恢复前提供校验与预览。

### P1 —— 加深洞察（决定留存与差异化）

5. **资产配置视图。** 按类别 / 成员 / 币种的占比，以及配置随时间的漂移。与多选走势筛选天然互补。
6. **净值变动归因。** 把已有的「变动原因」聚合成「**净值涨了 X：存下来 A、投资赚了 B、汇率影响 C**」。数据模型已经支持——这是最强的差异化点。
7. **目标与储蓄率。** 设定目标净值 / FIRE 数字并展示进度，按月计算主动储蓄率，给坚持更新一个理由。
8. **周期对比。** 净值与各类别的 MoM / YoY / YTD 变化。

### P2 —— 扩展边界（增长与广度）

9. **多设备 / 家庭协作同步。** 真正高价值的「家庭」场景是夫妻各自更新自己的账户。体量较大（账号体系、冲突合并、端到端加密），可先用「只读共享报告」过渡。
10. **报告与导出。** CSV / 月度 PDF 简报。
11. **更丰富的资产建模（可选）。** 对投资类账户支持「份额 × 单价」而非只填总额，放在高级开关后，避免偏离「简洁」。

### 如果只能做三件事

1. **应用锁** —— 信任底线。
2. **汇率自动获取 + 最小输入更新** —— 把更新摩擦降到最低，保住核心循环。
3. **净值变动归因** —— 把已有数据变成别人没有的洞察。

前两件保「用户会回来」，第三件保「用户离不开」。

## 架构

各层只能向下依赖：

- `lib/features/**` —— UI（页面/组件），可依赖 core + domain + l10n。
- `lib/core/**` —— providers、路由、主题、通知、备份、常量。
- `lib/data/**` —— Drift 数据库（`db/`）与仓储（`repositories/`）。
- `lib/domain/**` —— 纯 Dart：计算器、货币、预测、枚举（不引入 Flutter/Drift）。
- `lib/l10n/**` —— 生成的本地化文件。

关键约定：

- UI 不直接接触 `AppDatabase`，一律通过 Riverpod provider 暴露的仓储访问。
- 数据库派生状态以 `StreamProvider` 暴露，由 Drift 的 `watch()` / `dashboardChanges()` 驱动，保证净值等数据不会过期。
- domain 层保持与框架无关、可单元测试。

## 快速开始

前置条件：较新的 [Flutter](https://docs.flutter.dev/get-started/install) stable SDK（Dart 3.12+）。

```bash
flutter pub get          # 安装依赖
flutter gen-l10n         # 生成本地化文件
flutter run              # 在连接的设备/模拟器上运行
```

常用开发命令：

```bash
flutter analyze          # 静态分析（必须无报错）
flutter test             # 运行单元/组件测试
dart run build_runner build   # 修改表结构后重新生成 Drift 代码
```

新增用户可见文案时，编辑全部 `lib/l10n/app_*.arb` 文件（`app_en.arb` 为模板），然后运行 `flutter gen-l10n`。修改 `lib/data/db/tables.dart` 中的 Drift 表后，记得提升 `schemaVersion`、添加 `onUpgrade` 步骤并重新生成 Drift 代码。

### 新增语言

支持的语言由一个枚举统一管理，因此新增成本很低：

1. 复制 `lib/l10n/app_en.arb` 为 `lib/l10n/app_<code>.arb`（如 `app_fr.arb`）并翻译其中的值，将 `"@@locale"` 改为对应语言代码。
2. 在 `lib/domain/models/app_language.dart` 的 `AppLanguage` 枚举中新增一项，填入对应的 `code` 与该语言的本名 `nativeName`。
3. 运行 `flutter gen-l10n`。

「设置」中的语言选择器、`MaterialApp` 的 locale、`supportedLocales` 以及持久化的偏好都源自该枚举与生成的本地化文件，因此无需改动其他代码。
